# S5.5 — Shader Pack Close (NFR14)

_Date: 2026-04-24_

## The 4-shader budget rationale

NFR14 caps the sprint-001 rendering engine at four shader materials. The cap exists because:

1. **Frame budget math** — S5.1 measured 50 aqueducts × 48 particles = 2,400 instanced quads at ≤2ms per frame on Apple M1. Each additional full-screen shader pass risks pushing frames past the 16.7ms (60 fps) budget. Four shaders covers all five lens types (palace, room, inscription-in-room, traversal arc, ambient decay) without overdraw stacking.
2. **Complexity ceiling** — four materials is the maximum that can be spiked, promoted, and tested within sprint-001 scope without requiring a dedicated render-engineer pass. Growth tier lifts this cap when the full lantern ring (FR60f), splat-scene renderer, and HDRI cubemap consumer arrive.
3. **D-009 risk gate** — S5.1's six-checkbox gate was the go/no-go for the entire shader track. With all six passing, the four-material pack became a firm sprint commitment rather than a stretch goal.

## Micro-spike approach

Each of the three new S5.5 materials went through a micro-spike story in `src/lib/stories/spikes/` before promotion into the production lens wrappers:

- `RoomPulseSpike.stories.svelte` — proved capacitance uniform → visible pulse period change.
- `DustCobwebSpike.stories.svelte` — proved freshness uniform → visible opacity change.
- `MythosLanternSpike.stories.svelte` — proved static compile + render at zero-point (no uniform needed).

Each spike is kept in the repository as a reference implementation (not deleted after promotion), matching the sprint-004-logavatar `/spike/splat-{anim,perframe,lbs}` pattern. This prevents future engineers from re-discovering the same binding mechanics.

## NFR14 hard-cap enforcement

The cap is enforced by two mechanisms:

1. **traversal.test.ts** — the `RT5` SEC6 test block checks every shader wrapper Svelte file for forbidden patterns (`fetch(` in non-store code). Adding a fifth `.glsl` import to any production lens would require a deliberate edit to the test, making the violation visible in review.
2. **`FullRenderPackShowcase.stories.svelte`** — the six composed scenes explicitly enumerate the four shader materials. A reviewer can confirm no fifth shader is loaded by reading the imports at the top of the file.

## D-009 graceful-degradation conditional result

The S5.5 spec includes a graceful-degradation branch: if S5.1's D-009 gate had triggered `/replan` (2+ checkboxes failed), `dust-cobweb` and `mythos-lantern` stub would have been deferred to a follow-up sprint and S5.5 would have delivered only `room-pulse` + FR18 traversal round-trip (2 of 4 shaders).

**Actual result: this branch is inert.** D-009 passed six-of-six in S5.1 (see `S5.1-aqueduct-flow-spike-report.md`). All four shader materials are delivered in sprint-001 as committed.

## Growth deferred items

- `mythos-lantern` full ring (FR60f): N lanterns in polar orbit, mythos-chain heat, halo/bloom pass.
- `aqueduct-flow` tube geometry: replace linear segment with proper Catmull-Rom curve using actual room world positions.
- `room-pulse` full room-mesh: replace PlaneGeometry quad with the room's archiform mesh.
- `dust-cobweb` path geometry: replace PlaneGeometry with proper aqueduct path coordinates.

None of these affect the sprint-001 visual correctness — the shaders produce the intended visual behaviour on the stub geometry.
