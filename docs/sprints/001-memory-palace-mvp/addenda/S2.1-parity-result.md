## API notes for S2.3 implementors

- kuzu-wasm@0.11.3 browser (async) build uses `qr.getAllObjects()` not `qr.getAll()` — the two variants have different QueryResult APIs.
- Vector extension is bundled in kuzu-wasm WASM; `INSTALL VECTOR` / `LOAD EXTENSION VECTOR` are **not** needed in the browser (already bundled). They are only required for @ladybugdb/core native.
- Worker path must be set before first DB operation: `kuzu.setWorkerPath('/kuzu_wasm_worker.js')`.
- `kuzu_wasm_worker.js` must be served same-origin (copy to `static/` via `bun run bootstrap` per S2.3 AC1).
